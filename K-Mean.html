Tujuan dan Fungsi Klaster (Clustering)#

Clustering adalah salah satu teknik utama dalam unsupervised learning yang bertujuan untuk mengelompokkan data berdasarkan kemiripan atau kedekatan karakteristik tanpa menggunakan label atau target output. Klastering sering digunakan dalam data mining, analisis data eksploratif, segmentasi pasar, dan banyak bidang lain.

ğŸ¯ Tujuan Klastering#

  1. Mengelompokkan Data Serupa Tujuan utama klastering adalah mengelompokkan data ke dalam beberapa kelompok (klaster) sedemikian rupa sehingga:

    • Data dalam satu klaster mirip satu sama lain (intra-cluster similarity tinggi).

    • Data antar klaster berbeda jauh (inter-cluster dissimilarity tinggi).

  2. Reduksi Kompleksitas Data Dengan membagi data ke dalam klaster, kita bisa menyederhanakan analisis data, terutama saat bekerja dengan data besar dan kompleks.

  3. Mengidentifikasi Pola atau Struktur Tersembunyi Klastering dapat membantu mengungkap pola tersembunyi dalam data yang mungkin tidak tampak secara langsung.

  4. Segmentasi Dalam dunia bisnis dan pemasaran, klastering digunakan untuk membagi pelanggan ke dalam segmen berdasarkan perilaku atau atribut tertentu (misalnya: segmentasi pelanggan, segmentasi wilayah, dll).

ğŸ”¹ Penghapusan Kolom Class#

Karena data Iris sudah memiliki label (Class) yang menunjukkan kategori bunga (Iris-setosa, Iris-versicolor, dan Iris-virginica), maka:

  • Kolom Class harus dihapus terlebih dahulu sebelum dilakukan proses clustering atau normalisasi.

  • Ini penting karena Class adalah informasi target (ground truth), dan tidak boleh ikut dihitung dalam proses unsupervised learning seperti K-Means atau saat melakukan normalisasi.

ğŸ”¹ Normalisasi dengan MinMaxScaler#

Setelah penghapusan Class, fitur numerik seperti 'sepal length', 'sepal width', 'petal length', dan 'petal width' dinormalisasi menggunakan Min-Max Scaler, yang bekerja dengan rumus:

\[ x_{\text{scaled}} = \frac{x - x_{\text{min}}}{x_{\text{max}} - x_{\text{min}}} \]

ğŸ“Œ Keterangan:#

  • \(x\): nilai asli dari suatu fitur (misalnya: panjang kelopak = 4.7)

  • \(x_{\text{min}}\): nilai minimum dari fitur tersebut di seluruh dataset

  • \(x_{\text{max}}\): nilai maksimum dari fitur tersebut di seluruh dataset

  • \(x_{\text{scaled}}\): nilai hasil normalisasi yang berada dalam rentang [0, 1]

Penjelasannya:

  • Tujuan: menyamakan skala antar fitur agar tidak ada fitur yang mendominasi proses clustering karena memiliki nilai lebih besar.

  • Hasil: Semua nilai fitur akan berada di dalam rentang 0 hingga 1.

  • Kapan digunakan: Min-Max cocok digunakan jika tidak ada outlier ekstrem dan ingin menjaga distribusi asli fitur.

Langkah Langkah Algoritma K-Means#

  1. Tentukan jumlah cluster Disini saya akan mencoba dengan data yang ditampilkan membaginya dengan 2,3,4 klaster

  2. Pilih K centroid awal secara acak setiap klaster memiliki 1 centroid

  3. Hitung jarak setiap data ke tiap centroid gunakan rumus Encludian Distance $\( d(\mathbf{x}, \mathbf{c}) = \sqrt{ \sum_{i=1}^{n} (x_i - c_i)^2 } \)$

Penjelasan:#

  • \(\mathbf{x} = (x_1, x_2, \ldots, x_n)\): vektor data.

  • \(\mathbf{c} = (c_1, c_2, \ldots, c_n)\): vektor centroid.

  • \(n\): jumlah dimensi atau fitur.

  • \(d(\mathbf{x}, \mathbf{c})\): jarak Euclidean antara data dan centroid.

  1. Kelompokan data dengan centroid terdekat

  2. Hitung ulang centroid dari rata rata anggota kluster $\( \mu_k = \frac{1}{N_k} \sum_{x_i \in C_k} x_i \)$

âœ… Penjelasan:#

  • $\mu_k$ Menyatakan centroid (titik pusat) dari klaster ke-$k$.

  • $N_k$ Jumlah data (anggota) yang terdapat dalam klaster ke-$k$.

  • $x_i \in C_k$ Menyatakan bahwa $x_i$ adalah titik data yang termasuk ke dalam klaster $C_k$.

  • $\sum_{x_i \in C_k} x_i$ Menyatakan jumlah (penjumlahan vektor) dari seluruh titik data yang berada di dalam klaster $C_k$.

  • $\frac{1}{N_k}$ Digunakan untuk menghitung rata-rata dari semua anggota dalam klaster tersebut.

  1. Ulangi langkah 3-5 hingga konvergen(tidak ada perubahan signifikan)

ğŸ“˜ Apa itu PCA (Principal Component Analysis)?#

PCA atau Principal Component Analysis adalah metode reduksi dimensi yang digunakan dalam machine learning dan data analysis untuk menyederhanakan data berdimensi tinggi tanpa kehilangan informasi penting secara signifikan.

ğŸ¯ Tujuan PCA:#

  • Mengurangi dimensi data (fitur) menjadi lebih sedikit agar:

    • Analisis menjadi lebih mudah dan cepat.

    • Visualisasi data (misalnya ke 2D atau 3D) menjadi mungkin.

  • Menghilangkan redundansi (fitur yang saling berkorelasi).

  • Menangkap variansi terbesar dalam data dengan jumlah komponen yang lebih sedikit.

ğŸ› ï¸ Cara Kerja PCA (Secara Ringkas):#

Berikut adalah rumus utama dalam Principal Component Analysis (PCA)

1. Standarisasi Data#

Setiap fitur \(x_j\) dari data distandarisasi:

\[ z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j} \]

  • \(x_{ij}\): nilai fitur ke-\(j\) dari data ke-\(i\)

  • \(\mu_j\): rata-rata fitur ke-\(j\)

  • \(\sigma_j\): standar deviasi fitur ke-\(j\)

2. Kovarian Matriks#

Hitung matriks kovarian \(\mathbf{C}\) dari data yang telah distandarisasi:

\[ \mathbf{C} = \frac{1}{n-1} \mathbf{Z}^\top \mathbf{Z} \]

  • \(\mathbf{Z}\): matriks data hasil standarisasi

  • \(n\): jumlah sampel

3. Eigenvalue dan Eigenvector#

Cari eigenvalue dan eigenvector dari matriks kovarian:

\[ \mathbf{C} \mathbf{v} = \lambda \mathbf{v} \]

  • \(\lambda\): eigenvalue (mengukur variansi)

  • \(\mathbf{v}\): eigenvector (arah komponen utama)

4. Proyeksi ke Komponen Utama#

Proyeksikan data ke komponen utama:

\[ \mathbf{Z}_{\text{baru}} = \mathbf{Z} \mathbf{W} \]

  • \(\mathbf{W}\): matriks yang terdiri dari \(k\) eigenvector utama

  • \(\mathbf{Z}_{\text{baru}}\): data dalam ruang berdimensi lebih rendah (misalnya 2D)

from sklearn.cluster import KMeans
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt
import pandas as pd
import numpy as np

# Ambil fitur numerik
val_X = df[['petal length', 'petal width', 'sepal length', 'sepal width']].values

# KMeans model untuk 2 klaster
k_means_C2 = KMeans(
    n_clusters=2,
    init='k-means++',
    n_init=5,
    max_iter=400,
    tol=0.0001,
    random_state=42,
    algorithm='lloyd'
)

# Latih model
k_means_C2.fit(val_X)

# Ambil label dan centroid
labelC2 = k_means_C2.labels_
centroidsC2 = k_means_C2.cluster_centers_

# Tampilkan tabel ID dan klaster
df_clustered = df.copy()
df_clustered['Cluster'] = labelC2
tabel_klaster = df_clustered.reset_index()[['index', 'Cluster']]
tabel_klaster.columns = ['ID', 'Cluster']
print(tabel_klaster.to_string(index=False))

# Reduksi dimensi dengan PCA
pca = PCA(n_components=2)
val_X_pca = pca.fit_transform(val_X)
centroidsC2_pca = pca.transform(centroidsC2)

# Plot hasil klaster
plt.figure(figsize=(8, 6))
colors = ['red', 'blue',]

for i in range(2):  # Karena k=2

# Plot centroid

    s=200,

    edgecolor='black',
    marker='X',
    label='Centroid'
)

Evaluasi Klastering menggunakan Inersia#

Setelah kita melakukan klastering 2 klaster dengan k-means kita bisa melakukan evaluasi hasil klasteringnya dengan Inersia.

ğŸ“˜ Pengertian Inersia dalam K-Means Clustering#

Inersia (Inertia) dalam konteks K-Means Clustering adalah ukuran seberapa rapat data dalam setiap klaster terhadap centroid-nya masing-masing.

Secara lebih formal:

Inersia adalah jumlah kuadrat jarak dari setiap titik data ke centroid (titik pusat) klaster tempat data tersebut tergabung.

ğŸ” Fungsi Inersia:#

  • Menilai kualitas klastering: semakin kecil nilai inersia, semakin rapat data dalam klaster (semakin baik).

  • Digunakan untuk menentukan jumlah klaster optimal menggunakan metode Elbow: titik di mana penurunan inersia mulai melambat secara signifikan adalah jumlah klaster terbaik.

ğŸ’¡ Contoh:#

Jika kamu mengelompokkan bunga iris berdasarkan ukuran kelopak dan sepalnya, maka inersia menunjukkan seberapa dekat ukuran masing-masing bunga terhadap pusat dari kelompok (klaster) tempat dia masuk.

ğŸ§® Rumus Inertia K-Means (Total Within-Cluster Sum of Squares):#

\[ \text{Inertia} = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2 \]

ğŸ“˜ Keterangan:#

  • \(K\): jumlah klaster

  • \(C_k\): himpunan data dalam klaster ke-\(k\)

  • \(x_i\): titik data ke-\(i\) dalam klaster \(C_k\)

  • \(\mu_k\): centroid (rata-rata) dari klaster \(k\)

  • \(\|x_i - \mu_k\|^2\): kuadrat jarak Euclidean antara \(x_i\) dan centroid-nya

Evaluasi klastering menggunakan silhouette#

Selain menggunakan inersia kita juga bisa menggunakan silhouette untuk evaluasi 2 klaster dari K-means yang ditentukan sebelumnya

ğŸ“˜ Pengertian Silhouette Score#

Silhouette Score adalah metrik evaluasi untuk menilai kualitas klastering. Metrik ini mengukur seberapa mirip suatu data dengan klasternya sendiri dibandingkan dengan klaster lain.

Nilai Silhouette berada dalam rentang:

  • +1: Titik sangat cocok dengan klasternya.

  • 0: Titik berada di batas antara dua klaster.

  • âˆ’1: Titik kemungkinan salah klaster.

ğŸ” Tujuan Silhouette Score:#

  • Menilai apakah pemisahan antar klaster sudah baik.

  • Digunakan untuk memilih jumlah klaster optimal.

ğŸ§® Rumus dalam LaTeX:#

Jika:

  • \(a(i)\): Rata-rata jarak antara titik \(i\) dan semua titik lain dalam klaster yang sama.

  • \(b(i)\): Rata-rata jarak antara titik \(i\) dan semua titik dalam klaster terdekat yang berbeda.

Maka:

\[ s(i) = \frac{b(i) - a(i)}{\max\{a(i), b(i)\}} \]

ğŸ“Š Interpretasi:#

  • \(s(i) \approx 1\): Titik berada di dalam klaster yang tepat.

  • \(s(i) \approx 0\): Titik berada di batas antar klaster.

  • \(s(i) < 0\): Titik lebih dekat ke klaster lain daripada klasternya sendiri â†’ kemungkinan salah klaster.

Jika ingin menghitung rata-rata Silhouette Score untuk seluruh data:

\[ \text{Silhouette Score} = \frac{1}{n} \sum_{i=1}^{n} s(i) \]

Dengan \(n\) adalah jumlah total data.

Evaluasi K-means dengan Metode Elbow#

setelah kita melakukan klastering dengan 2,3,4 klaster lalu mengevaluasi dengan Inersia sebagaimana perlu mengevaluasi kembali nilai inersianya dengan metode elbow untuk mencari klaster yang paling optimal

ğŸ“˜ Penjelasan tentang Metode Elbow dalam K-Means Clustering#

Metode Elbow adalah teknik grafis yang digunakan untuk menentukan jumlah klaster optimal dalam algoritma K-Means Clustering.

ğŸ§  Inti dari Metode Elbow:#

Tujuannya adalah mencari nilai \(k\) (jumlah klaster) terbaik, di mana penambahan klaster tidak memberikan penurunan signifikan terhadap inertia (jumlah kuadrat jarak titik ke centroid).

ğŸ“‰ Langkah-langkah Metode Elbow:#

  1. Jalankan K-Means untuk beberapa nilai \(k\) (misal dari 1 sampai 10).

  2. Catat nilai inertia untuk tiap \(k\).

  3. Buat grafik:

    • Sumbu X = jumlah klaster \(k\)

    • Sumbu Y = nilai inertia

  4. Cari titik â€œtekukanâ€ (elbow) di grafik.

    • Elbow point adalah nilai \(k\) di mana penurunan inertia mulai melambat tajam.

    • Di titik ini, penambahan klaster sudah tidak terlalu meningkatkan kualitas klastering.

âœ… Kesimpulan:#

Metode Elbow membantu menghindari overfitting (terlalu banyak klaster) dan underfitting (terlalu sedikit klaster), dengan memilih jumlah klaster yang efisien dan optimal secara visual berdasarkan kurva inertia.

from sklearn.cluster import KMeans
import numpy as np
import matplotlib.pyplot as plt

Berikut adalah penjelasan lengkap dari perhitungan fungsi objektif \(J\) pada algoritma Fuzzy C-Means berdasarkan data dan centroid yang telah diberikan:

ğŸ”¢ Tujuan Perhitungan#

Fungsi objektif \(J\) digunakan untuk mengukur seberapa baik pembagian klaster dilakukan. Semakin kecil nilai \(J\), semakin baik pemisahan data ke dalam klaster-klaster tersebut.

ğŸ“Œ Rumus Fungsi Objektif \(J\):#

\[ J = \sum_{i=1}^{n} \left( u_{i1}^m \cdot d_{i1}^2 + u_{i2}^m \cdot d_{i2}^2 \right) \]

Dimana:

  • \(u_{i1}, u_{i2}\): nilai keanggotaan data ke klaster 1 dan 2

  • \(m\): parameter fuzziness (biasanya 2)

  • \(d_{i1}, d_{i2}\): jarak Euclidean dari data ke centroid klaster 1 dan 2

  • \(d_{ij} = \sqrt{(x_i - c_{jx})^2 + (y_i - c_{jy})^2}\)

ğŸ’¡ Langkah-Langkah Perhitungan#

  1. Gunakan nilai keanggotaan fuzzy dari iterasi sebelumnya:

    • Misalnya, untuk data pertama: \(u_1 = 0.4344\), maka \(u_2 = 1 - 0.4344 = 0.5656\)

  2. Gunakan centroid hasil iterasi:

    • \(\text{Centroid c1} = (3.8610, 4.8610)\)

    • \(\text{Centroid c2} = (3.5079, 4.5079)\)

  3. Hitung jarak Euclidean dari setiap titik data ke masing-masing centroid:

    \[ d_1 = \sqrt{(x - c1_x)^2 + (y - c1_y)^2} \]
    \[ d_2 = \sqrt{(x - c2_x)^2 + (y - c2_y)^2} \]

  4. Hitung kontribusi masing-masing data ke nilai \(J\) dengan:

    \[ u_1^2 \cdot d_1^2 + u_2^2 \cdot d_2^2 \]

  5. Jumlahkan semua kontribusi dari seluruh data.

Hitung Pusat Klaster awal#

Dalam algoritma Fuzzy C-Means (FCM), pusat klaster (centroid) dihitung dengan mempertimbangkan derajat keanggotaan fuzzy dari setiap data terhadap masing-masing klaster. Ini berbeda dari K-Means biasa yang hanya menggunakan anggota tetap (hard clustering).

ğŸ“Œ Rumus Perhitungan Pusat Klaster (Centroid)#

Untuk klaster ke-ğ‘—, pusat klaster dihitung sebagai:

\[ v_j = \frac{\sum_{i=1}^{n} u_{ji}^m \cdot x_i}{\sum_{i=1}^{n} u_{ji}^m} \]

ğŸ“˜ Makna Simbol:#

Simbol

Makna

\(v_j\)

Vektor pusat klaster ke-ğ‘— (centroid ke-ğ‘—)

\(x_i\)

Vektor fitur (data) ke-ğ‘–

\(u_{ji}\)

Derajat keanggotaan data ke-ğ‘– terhadap klaster ke-ğ‘—

\(m\)

Parameter fuzziness (biasanya \(m = 2\))

\(n\)

Jumlah total data

\(\sum_{i=1}^{n}\)

Penjumlahan dari semua data ke-ğ‘–

ğŸ”„ Penjelasan Konsep:#

  • Derajat keanggotaan fuzzy \(u_{ji}\) menyatakan sejauh mana data ke-ğ‘– termasuk dalam klaster ke-ğ‘—, dengan nilai antara 0 sampai 1.

  • Derajat keanggotaan tersebut dipangkatkan dengan \(m\) (fuzziness) untuk memberikan kontribusi non-linear terhadap centroid. Semakin besar \(m\), semakin â€œfuzzyâ€ atau tidak tegas pembagian klasternya.

  • Pusat klaster dihitung sebagai rata-rata berbobot (weighted average) dari semua data, di mana bobotnya adalah \(u_{ji}^m\).

update Derajat Keanggotaannya#

Setelah menghitung pusat klaster (\(v_j\)), langkah selanjutnya dalam algoritma Fuzzy C-Means adalah memperbarui derajat keanggotaan fuzzy untuk setiap data ke setiap klaster.

ğŸ“˜ Rumus Perbaruan Derajat Keanggotaan#

Untuk data ke-ğ‘– dan klaster ke-ğ‘—:

\[ u_{ji} = \left( \sum_{k=1}^{c} \left( \frac{||x_i - v_j||}{||x_i - v_k||} \right)^{\frac{2}{m-1}} \right)^{-1} \]

ğŸ“Œ Makna Simbol:#

Simbol

Makna

\(u_{ji}\)

Derajat keanggotaan data ke-ğ‘– terhadap klaster ke-ğ‘—

\(x_i\)

Vektor fitur dari data ke-ğ‘–

\(v_j\)

Pusat klaster ke-ğ‘— (centroid ke-ğ‘—)

\(m\)

ğŸ’¡ Penjelasan Intuitif:#

  • Formula ini menghitung seberapa dekat data ke-ğ‘– dengan klaster ke-ğ‘— dibandingkan dengan semua klaster lain.

  • Jika \(x_i\) lebih dekat ke \(v_j\) daripada ke pusat klaster lainnya, maka nilai \(u_{ji}\) akan lebih besar.

  • Semakin jauh data dari suatu klaster, semakin kecil keanggotaannya terhadap klaster tersebut.

ğŸ” Catatan:#

  • Derajat keanggotaan selalu berada dalam rentang \([0, 1]\).

  • Untuk setiap data \(i\), jumlah seluruh \(u_{ji}\) terhadap semua klaster \(j\) adalah 1:

\[ \sum_{j=1}^{c} u_{ji} = 1 \quad \forall i \]
# Hitung ulang derajat keanggotaan berdasarkan pusat cluster
m = 2.0
n_data = X_scaled.shape[0]
n_clusters = cntr.shape[0]

# Buat matriks kosong untuk u_ij
u_updated = np.zeros((n_clusters, n_data))

for j in range(n_data):  # Untuk setiap data
    x_j = X_scaled[j]
    for i in range(n_clusters):  # Untuk setiap cluster
        numerator = np.linalg.norm(x_j - cntr[i])  # ||x_j - v_i||
        if numerator == 0:
            u_updated[:, j] = 0
            u_updated[i, j] = 1  # Jika x_j = v_i, maka full membership ke cluster i
            break
        denom = 0
        for k in range(n_clusters):
            dist = np.linalg.norm(x_j - cntr[k])
            denom += (numerator / dist) ** (2 / (m - 1))
        u_updated[i, j] = 1 / denom

Hitung ulang pusat klaster dan iterasikan sampai konvergen#

ğŸ“˜ Rumus Menghitung Pusat Klaster (Centroid):#

Untuk klaster ke-ğ‘—:

\[ v_j = \frac{\sum_{i=1}^{n} \left( u_{ji} \right)^m \cdot x_i}{\sum_{i=1}^{n} \left( u_{ji} \right)^m} \]

ğŸ“Œ Makna Simbol:#

Simbol

Makna

\(v_j\)

Pusat klaster ke-ğ‘— (centroid)

\(u_{ji}\)

Derajat keanggotaan data ke-ğ‘– terhadap klaster ke-ğ‘—

\(m\)

Parameter fuzziness

\(x_i\)

Vektor fitur dari data ke-ğ‘–

\(n\)

Jumlah total data

ğŸ’¡ Penjelasan Intuitif:#

  • Pusat klaster dihitung sebagai rata-rata berbobot fuzzy dari semua data.

  • Bobot tiap data terhadap suatu klaster adalah \(u_{ji}^m\) â€” makin besar keanggotaannya, makin besar pengaruhnya terhadap pusat klaster tersebut.

ğŸ” Iterasi Sampai Konvergen:#

Langkah-langkah:

  1. Inisialisasi derajat keanggotaan \(u_{ji}\) secara acak.

  2. Hitung pusat klaster \(v_j\) dengan rumus di atas.

  3. Update \(u_{ji}\) dengan rumus keanggotaan:

    \[ u_{ji} = \left( \sum_{k=1}^{c} \left( \frac{||x_i - v_j||}{||x_i - v_k||} \right)^{\frac{2}{m-1}} \right)^{-1} \]

  4. Cek konvergensi, misalnya dengan:

    \[ \text{max\_change} = \max_j ||v_j^{(t)} - v_j^{(t-1)}|| \]

    Jika perubahan maksimum antar pusat klaster < tolerance (misal 1e-6), stop iterasi.

  5. Jika belum konvergen, ulang dari langkah 2.

  • Pusat-pusat klaster tidak berubah signifikan lagi.

  • Algoritma telah menemukan posisi klaster optimal menurut kriteria FCM.
