Decision Tree#

Pohon Keputusan(Decision Tree adalah salah satu metode dalam penambangan data yang digunakan untuk klasifikasi dan regresi. Model ini sangat populer karena strukturnya yang intuitif, menyerupai diagram alir atau pohon, sehingga mudah dipahami bahkan oleh orang non-teknis.

Secara sederhana, Decision Tree memetakan keputusan-keputusan yang mungkin diambil beserta konsekuensinya. Setiap simpul (node) dalam pohon merepresentasikan sebuah â€œtesâ€ terhadap suatu atribut (fitur), setiap cabang (branch) merepresentasikan hasil dari tes tersebut, dan setiap daun (leaf node) merepresentasikan label kelas (keputusan akhir).

Bagaimana Cara Kerjanya?

Algoritma Decision Tree bekerja dengan cara mempartisi atau membagi data secara berulang-ulang berdasarkan fitur/atribut tertentu. Tujuannya adalah untuk membuat sub-grup data yang se-â€œmurniâ€ mungkin. â€œMurniâ€ di sini berarti semua anggota dalam sub-grup tersebut memiliki kelas atau hasil yang sama.

Proses utamanya adalah:

  • Pilih Atribut Terbaik: Algoritma dimulai dari Simpul Akar (Root Node) yang berisi seluruh dataset. Kemudian, ia mencari atribut mana yang paling baik dalam memisahkan data menjadi kelompok-kelompok yang paling murni. Untuk mengukur â€œkemurnianâ€ ini, metrik seperti Gini Impurity atau Information Gain (Entropy) digunakan.

  • Lakukan Pemisahan (Splitting): Dataset dibagi menjadi beberapa subset berdasarkan nilai dari atribut yang terpilih. Setiap subset ini akan menjadi cabang baru.

  • Ulangi Proses: Proses nomor 1 dan 2 diulangi untuk setiap subset (yang kini menjadi simpul baru) sampai salah satu kondisi berikut terpenuhi:

  1. Semua data dalam simpul sudah memiliki kelas yang sama (sudah murni).

  2. Tidak ada lagi atribut yang bisa digunakan untuk memisahkan data.

  3. Pohon sudah mencapai kedalaman maksimum yang ditentukan sebelumnya untuk mencegahnya menjadi terlalu rumit.

  4. Simpul yang tidak bisa dibagi lagi disebut Simpul Daun (Leaf Node) dan berisi keputusan atau prediksi akhir.

Istilah Penting dalam Decision Tree

  • Simpul Akar (Root Node): Simpul paling atas yang mewakili seluruh populasi atau dataset.

  • Simpul Keputusan (Decision Node): Simpul yang memiliki dua atau lebih cabang, merepresentasikan sebuah tes pada suatu atribut.

  • Simpul Daun (Leaf Node): Simpul akhir yang tidak bisa dipecah lagi dan menunjukkan hasil/keputusan akhir (label kelas).

  • Pemisahan (Splitting): Proses membagi sebuah simpul keputusan menjadi sub-simpul.

  • Pemangkasan (Pruning): Proses menghilangkan cabang dari pohon keputusan untuk mengurangi kompleksitas dan mencegah overfitting. Overfitting adalah kondisi di mana model terlalu â€œmenghafalâ€ data latih sehingga performanya buruk saat dihadapkan pada data baru.

Langkah-langkah Decision Tree#

  1. Menghitung Entropy Awal

Secara sederhana, Entropy adalah ukuran ketidakpastian, keacakan, atau ketidakteraturan dalam sekumpulan data. Semakin tinggi nilai Entropy, semakin acak dan tidak teratur data tersebut. Sebaliknya, semakin rendah nilai Entropy, semakin teratur dan mudah ditebak data tersebut.

Rumus Entropy#

Rumus untuk menghitung Entropy adalah:

\[Entropy(S) = \sum_{i=1}^{c} -p_i \log_2(p_i)\]

Di mana:

  • \(S\) = Keseluruhan dataset.

  • \(c\) = Jumlah kelas unik (dalam kasus ini ada 3: Tidak Lulus, Lulus, Lulus dengan Pujian).

  • \(p_i\) = Proporsi dari kelas ke-i dalam dataset(jumlah data kelas ke-i dibagi keseluruhan data).

  1. Urutkan datanya(jika berupa numerik) Pastikan untuk setiap fitur sudah berupa data yang terurut

  2. ğŸ“Œ Penjelasan Threshold dalam Decision Tree

Threshold dalam Decision Tree adalah batas nilai yang digunakan untuk memisahkan data pada atribut numerik (kontinu), seperti Jam Belajar atau Nilai Ujian. Tujuannya adalah membagi data ke dalam dua kelompok yang sehomogen mungkin terhadap label atau kelas (misalnya: Tidak Lulus, Lulus, Lulus dengan Pujian).

Jika kita bisa membagi data dengan threshold sehingga:

  • satu sisi didominasi oleh satu kelas tertentu (contoh: semua â€œTidak Lulusâ€),

  • dan sisi lainnya oleh kelas lain (contoh: semua â€œLulusâ€),

â€¦maka model lebih akurat dan pohonnya lebih sederhana.

ğŸ§® Rumus Threshold#

Threshold dicari di titik tengah antara dua nilai berurutan yang memiliki kelas berbeda. Ini disebut sebagai midpoint threshold.

âœ… Rumus Threshold:#

Jika kamu memiliki dua nilai vâ‚ dan vâ‚‚ (urutan setelah sorting), maka:

\[ \text{Threshold} = \frac{v_1 + v_2}{2} \]

Dengan syarat bahwa nilai vâ‚ dan vâ‚‚ berasal dari kelas yang berbeda.

ğŸ§‘â€ğŸ« Contoh:#

Misalkan data Nilai Ujian yang sudah diurutkan:

40(A), 45(A), 50(A), 55(A), 59(B), 65(B), ...

Lihat perubahan kelas dari A â†’ B di antara 55 dan 59. Maka:

\[ \text{Threshold} = \frac{55 + 59}{2} = 57 \]

Threshold ini diuji untuk split: nilai < 57 vs nilai â‰¥ 57.

  1. hitung entropy dari masing masing thresholdnya

Berikut adalah penjelasan lengkap dan rumus dari proses menghitung entropy split dalam decision tree (C4.5 atau ID3), berdasarkan contoh yang kamu berikan:

âœ… APA ITU ENTROPY SPLIT?#

Entropy Split digunakan untuk mengukur ketidakpastian rata-rata dari pembagian data ke dalam dua subset berdasarkan threshold tertentu dari sebuah atribut.

Tujuan dari menghitung entropy split adalah untuk mengetahui seberapa â€œbersihâ€ pembagian data berdasarkan threshold tertentu. Semakin kecil nilai entropy split, semakin baik pembagiannya.

âœ… LANGKAH-LANGKAH + RUMUS#

1. Tentukan Threshold dan Bagi Data#

Misalnya, kita pilih atribut Nilai Ujian, lalu threshold-nya adalah 57.

Maka:

  • S â‰¤ 57 â†’ Data dengan nilai ujian â‰¤ 57

  • S > 57 â†’ Data dengan nilai ujian > 57

Contoh:

  • S â‰¤ 57 â†’ 4 data (semua dari kelas A)

  • S > 57 â†’ 11 data (6 dari kelas B, 5 dari kelas C)

2. Hitung Entropy Masing-Masing Subset#
ğŸ”¹ a. Entropy untuk S â‰¤ 57#

Karena semua data di subset ini berasal dari satu kelas (kelas A), maka entropinya 0 (tidak ada ketidakpastian):

\[ \text{Entropy}(S_1) = 0 \]

ğŸ”¹ b. Entropy untuk S > 57#

Ada dua kelas: B dan C.

  • Jumlah total data: 11

  • Jumlah data kelas B: 6 â†’ \(p_B = \frac{6}{11}\)

  • Jumlah data kelas C: 5 â†’ \(p_C = \frac{5}{11}\)

\[ \text{Entropy}(S_2) = -p_B \cdot \log_2(p_B) - p_C \cdot \log_2(p_C) \]
\[ = -\left( \frac{6}{11} \log_2 \frac{6}{11} + \frac{5}{11} \log_2 \frac{5}{11} \right) \]

Hitung:

\[ \frac{6}{11} â‰ˆ 0.545,\quad \log_2(0.545) â‰ˆ -0.874 \]
\[ \frac{5}{11} â‰ˆ 0.455,\quad \log_2(0.455) â‰ˆ -1.137 \]
\[ = -\left( 0.545 \cdot (-0.874) + 0.455 \cdot (-1.137) \right) = 0.476 + 0.518 = 0.994 \]

  1. Hitung Entropy Split Total

\[ \text{Entropy}_{split} = \frac{|S_1|}{|S|} \cdot \text{Entropy}(S_1) + \frac{|S_2|}{|S|} \cdot \text{Entropy}(S_2) \]
\[ = \frac{4}{15} \cdot 0 + \frac{11}{15} \cdot 0.994 = 0.7293 \]

  1. ** hitung information gainnya** Baik! Sekarang kita akan membahas langkah ke-5 dalam membangun decision tree, yaitu menghitung Information Gain (IG). Di bawah ini penjelasan lengkapnya:

âœ… Apa itu Information Gain?#

Information Gain (IG) mengukur seberapa besar pengurangan entropi yang diperoleh dengan membagi data berdasarkan fitur tertentu.

Artinya: Semakin tinggi nilai IG, semakin baik fitur tersebut dalam memisahkan kelas (misalnya, â€œTidak Lulusâ€, â€œLulusâ€, â€œLulus dengan Pujianâ€).

âœ… Rumus Information Gain#
\[ \text{Information Gain (IG)} = \text{Entropy Awal (S)} - \text{Entropy Split (S|A)} \]

Keterangan:

  • Entropy(S): Entropi seluruh dataset sebelum dibagi.

  • Entropy(S|A): Entropi rata-rata setelah data dibagi berdasarkan atribut A.

âœ… Langkah Perhitungan (Contoh Kasus)#

Misalkan:

  • Entropy awal seluruh dataset (sebelum split):

    \[ \text{Entropy}(S) = 1.566 \]

  • Entropy split setelah membagi berdasarkan atribut nilai ujian dengan threshold 57:

    \[ \text{Entropy}_{split} = 0.7293 \]
âœ… Hitung Information Gain#
\[ \text{IG} = 1.566 - 0.7293 = 0.8367 \]
âœ… Kesimpulan#

  • Nilai Information Gain = 0.8367

  • Karena nilai ini cukup tinggi, maka atribut â€œNilai Ujianâ€ dengan threshold 57 adalah pembagi yang baik.

  • Jika kamu membandingkan beberapa fitur, pilih fitur dengan IG tertinggi untuk menjadi node cabang utama pada decision tree.

  1. Pilih Atribut dengan Information Gain Tertinggi

Fitur dengan IG tertinggi dipilih sebagai atribut pembagi (node).

Threshold-nya digunakan untuk memisahkan data ke dalam cabang-cabang (misalnya, <= threshold dan > threshold).

ğŸ“Œ Contoh: Jika â€œNilai Ujianâ€ dengan threshold 57 memiliki IG tertinggi, maka:

Node decision tree â†’ â€œNilai Ujian <= 57?â€

Cabang kiri (yes): Data dengan nilai â‰¤ 57

Cabang kanan (no): Data dengan nilai > 57

  1. **buat cabang dan rekursif ulangi dari langkah 3

Untuk masing-masing cabang (hasil split):

a. Cek apakah entropi = 0 (pure): Jika ya â†’ node menjadi leaf (kelas mayoritas di data cabang).

Misalnya, jika semua datanya â€œTidak Lulusâ€, maka labelnya langsung = â€œTidak Lulusâ€.

b. Jika entropi â‰  0 (belum pure): Ulangi proses dari langkah 2 hingga 6 di subset data cabang itu:

Hitung entropy

Hitung IG semua fitur yang belum dipakai

Pilih IG tertinggi

Split lagi data

Lanjutkan hingga semua data terklasifikasi dengan baik atau fitur habis

  1. **Selesai Jika Semua Data Terklasifikasi

Decision tree selesai jika:

  • Semua node menjadi leaf (kelas sudah pure), atau

  • Tidak ada lagi fitur yang bisa digunakan

import pandas as pd
from sklearn.tree import DecisionTreeClassifier, export_graphviz
from sklearn.preprocessing import LabelEncoder
import graphviz

# 1. Menyiapkan Data
# Membuat DataFrame dari data terakhir yang Anda berikan.
data = {
    'Jam Belajar per Minggu': [2, 3, 3, 1, 4, 8, 6, 10, 5, 9, 12, 15, 11, 14, 13],
    'Nilai Ujian Sebelumnya': [55, 50, 45, 40, 59, 75, 65, 70, 80, 68, 85, 90, 92, 88, 82],
    'Status Kelulusan': [
        'Tidak Lulus', 'Tidak Lulus', 'Tidak Lulus', 'Tidak Lulus', 'Lulus',
        'Lulus', 'Lulus', 'Lulus', 'Lulus', 'Lulus',
        'Lulus dengan Pujian', 'Lulus dengan Pujian', 'Lulus dengan Pujian',
        'Lulus dengan Pujian', 'Lulus dengan Pujian'
    ]
}
df = pd.DataFrame(data)

# 2. Preprocessing Data
# Memisahkan fitur (X) dan target (y)
X = df[['Jam Belajar per Minggu', 'Nilai Ujian Sebelumnya']]
y = df['Status Kelulusan']

# Mengubah label target dari teks menjadi angka (0, 1, 2)
# karena model memerlukan input numerik.
le = LabelEncoder()
y_encoded = le.fit_transform(y)
# Mengetahui kelas asli dari label yang di-encode:
# 0: Lulus, 1: Lulus dengan Pujian, 2: Tidak Lulus
# (urutan berdasarkan abjad)
class_names_original = le.classes_

# 3. Membuat dan Melatih Model Decision Tree
# Membuat model. max_depth=3 berarti pohon tidak akan lebih dari 3 tingkat,
# untuk mencegah terlalu rumit dan mudah dibaca.
# criterion='entropy' agar sesuai dengan perhitungan manual kita.
model = DecisionTreeClassifier(criterion='entropy', max_depth=3, random_state=42)

# Melatih model dengan seluruh data
model.fit(X, y_encoded)

print("Model Decision Tree berhasil dilatih.")

# 4. Visualisasi Pohon Keputusan
# Mengekspor model ke dalam format DOT
dot_data = export_graphviz(
    model,
    out_file=None,
    feature_names=X.columns,
    class_names=class_names_original,
    filled=True,
    rounded=True,
    special_characters=True
)

# Menampilkan grafik di notebook
graph = graphviz.Source(dot_data)
print("\nVisualisasi Pohon Keputusan:")
# Untuk menyimpan gambar sebagai file PNG
# graph.render("decision_tree_kelulusan")
graph

Model Decision Tree berhasil dilatih.
